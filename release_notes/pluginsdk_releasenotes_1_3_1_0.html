    Plugin SDK v1.3.1.0 Release Notes

    This release focuses on changes to ensure that HS Platform v4.2 is ready for release.

    • Add internal method HsFeature.SetDisplayPriority() so that the HS Platform can properly populate the DisplayPriority field when delivering device/feature data to a plugin. (PSDK-193)
    • Add new flags to devices and events to allow controlling access to Amazon and Google individually. This is, by default, off. It is recommended to keep this an "opt in" process - meaning ask users before enabling access - to make sure users have a say in how their devices and events are exposed to these systems. (PSDK-204)
      • Add EMiscFlag.GoogleDiscoveryEnabled
      • Add EMiscFlag.AmazonDiscoveryEnabled
      • Add EEventFlag.GoogleDiscoveryEnabled
      • Add EEventFlag.AmazonDiscoveryEnabled
    • Fix ValueRange.IsValueInRange() so that it properly reports on values at both upper and lower ranges. (PSDK-196)
    • Fix an issue with AbstractHsDevice.Changes removing duplicate values unexpectedly. (PSDK-159)
    • Update build configurations to better support automated testing. Renamed Release and Debug configurations to HS-Release and HS-Debug, and added a new Release and Debug configuration.
